#### 1. 自动垃圾回收（GC）

- Go 内置垃圾回收器，自动管理内存生命周期。
- 开发者无需手动释放内存，降低内存泄漏和悬空指针风险。
- GC 机制不断优化，延迟更低，适合服务端场景。

#### 2. 丰富的内置类型

- 基础类型：整型、浮点型、字符串、布尔等
- 复合类型：数组、切片、map、结构体
- 并发类型：channel这些类型让 Go 在表达数据结构时非常自然。

#### 3. 多返回值

- 函数可以返回多个值，常用于返回结果 + 错误。
- 避免异常机制带来的复杂控制流。

#### 4. 明确的错误处理机制

- 使用 error 类型显式返回错误。
- 强调“错误即值”，让错误处理更可控、更清晰。

#### 5. 匿名函数与闭包

- 支持函数作为一等公民。
- 可用于回调、延迟执行、函数式编程等场景。

#### 6. 类型系统与接口

- 无继承，强调组合优于继承。
- 接口是隐式实现的，只要方法匹配即可。
- 减少模板代码，提高灵活性。

#### 7. 并发编程（Goroutine + Channel）

- Goroutine 是轻量级线程，由 Go runtime 调度。
- Channel 用于安全通信，实现 CSP 并发模型。
- 并发是 Go 的核心优势之一。

#### 8. 反射

- 可在运行时检查类型、字段、方法。
- 常用于 ORM、序列化、依赖注入等框架。

#### 9. 语言交互性

- 通过 cgo 与 C 语言交互。
- 可复用底层库或进行性能优化。
